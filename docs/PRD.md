2. 产品概述与定位

产品愿景：“让每一次随手摘录，都汇成个人的精神花期。”
产品定位：一个私有的、有审美温度的AI记忆管家，专为内容审美积累场景设计。

目标用户画像：日常大量碎片化输入，有收集癖，但厌恶手动分类，希望保留原始时间与来源的阅读爱好者/内容创作者。

总结来看，主要痛点有：
(1)内容分散，碎片化内容较多，无法按来源（播客、书籍、微博）统一浏览，或者按照类别进行分类
(2)修改内容后，缺少时序感，感受不到时间轴
(3)缺少回顾和关联，存了即埋掉

机会：大模型可以自动理解碎片内容、提取结构化信息，并发现隐性关联，完全能构建一个“第二大脑”式的记忆系统。

4. 产品目标与成功指标

产品目标：
朝花致力于在碎片化审美积累场景下，向用户交付三层核心价值：
（1）能够快速存入，让碎片收纳像呼吸一样自然。
（2）通过AI的关联推荐，结构化整合，触发用户自我认知的Aha时刻，让积累产生连接。
（3）让每一次积累都成为可回顾、可筛选、可生长的个人审美资产，建立长期复利价值。

5. 功能需求（按优先级 P0/P1/P2）
P0 核心闭环（Demo必须实现）

[拾花] 多模态碎片录入

支持文本直接粘贴、截图上传（调用OCR）

智能解析附言（来源、类型、期数）

[成卡] 自动化知识卡片生成

AI提取核心原文、来源信息、3-5个关键词标签

卡片包含不可变字段：原始采集时间（核心反痛点设计）

[花间] 历史卡片语义检索

基于向量相似度，推荐之前的相关花瓣

用户可确认建立双向链接（花径）

P1 增强体验

[花册] 个人知识库浏览

按时间线、来源、标签的筛选浏览

[花期] 定期回顾生成

每周/月自动生成“花期总结”，呈现你关注的主题演化

P2 未来探索

社会化分享与协作、多种导出格式、基于积累数据的个人风格分析

面试价值点：功能分优先级，证明你有迭代思维，知道MVP死守的核心是什么，不做大而全。

6. 交互与用户旅程（核心流程）
建议你用Mermaid流程图在这里画出来，或者手绘拍照嵌入

主流程：拾花

用户将一段播客金句 +" 来自《文化有限》Vol.xxx" 发给 Agent
Agent 解析输入，若为图片则转文字
Agent 生成卡片初稿并展示：“我帮你整理成了这样，要调整吗？”
之后Agent主动提示关联：“这和你3月存的《小王子》里那一句很呼应，要连起来吗？”
用户确认，存入花册，返回成功提示和时间标签
异常流：

来源解析不明确 → Agent反询问户，保留当前进度

OCR效果差 → 高亮存疑文字，请求输入原文

面试价值点：不仅要画正常流程，一定要画出异常处理和询问环节，这直接体现了你能考虑AI的不确定性，并且懂得把控制权交还用户。

7. 数据与记忆设计
数据核心实体：花卡 (Flower Card)

字段示例：id, raw_text, source_type(book/podcast/weibo等), source_name, author, created_at(不可变!), tags, ocr_text, summary, linked_card_ids

记忆机制：

短期：当前对话上下文

长期：结构化存储（SQLite）+ 向量存储（Chroma）

面试价值点：作为有工程背景的PM，你能定义数据结构，且强调不可变时间字段这个产品理念，非常出彩。

8. Agent 行为与提示词策略（架构层面的说明）
Agent角色：一位品味相投的私人图书馆管理员，耐心、有审美、不喧宾夺主。

核心规则：

永远保留用户原文

不确定时必须请求确认，不猜测

关联推荐时，说明关联理由

该部分会在后续 agent-design.md 详述，这里先给出角色定义。

9. 风险与护栏
版权风险：摘抄目的为个人学习欣赏，产品不提供公开分发功能，提示用户遵守合理使用。

内容安全：过滤明显违规内容

数据丢失：支持导出备份

面试价值点：展示你作为产品经理的责任边界意识，你不是只做一个酷炫功能，而是会考虑法律和伦理。
